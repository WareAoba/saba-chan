### 개요

해당 문서는 AI Agent들을 위해 작성되었다.
내 개인적인 희망 사항이나 TodoList를 위한 것이므로,
AI Agent의 적극적인 "읽기"는 권장하나, "수정"은 기본적으로 *불허*한다.

1. 모듈

모듈은 말 그대로 서버를 관리하는 방법론을 담은 지령서이다.
따라서 모듈에는 다음과 같은 사항이 명시되어야 한다:

 - 게임의 이름
 - 서버 파일의 위치(수정 가능)
 - 해당 서버에서 사용할 수 있는 명령어
 - 해당 서버의 조작 방식
 - 모듈의 버전
 - etc...

또한 파이썬으로 작성되는 모듈은, 통합 GUI 상에서 키보드와 마우스로 설정값을 제어하고,
저장이 필요한 값은 GUI의 config 저장 파일에 함께 보관되어야 한다.

2. API

API로 다음 기능의 구현이 필요하다 :

 - 디스코드 봇(명령어 감지) -> 코어 대몬 -> 서버 프로세스(실제 명령어 입력)

이 프로세스가 제대로 이루어져야 디스코드 봇으로서의 기능이 제대로 작동할 수 있을 것이다.

3. GUI

GUI 상에서는:

 - 간단한 조작은 마우스만으로(버튼 클릭, 스크롤바 등)
 - 상세한 명령어 직접 입력을 위한 CLI

해당 기능이 동작해야 한다.
물론 이것은 "게임에 따라 상이"하기 때문에,
모듈에 명시해둘 필요는 있지만, 중요한 건
어느 모듈을 쓰더라도 일관화된 UX를 제공하기 위해서
리액트의 컴포넌트를 가져다 쓰듯, 통일된 환경을 갖출 수 있도록 해야 한다.
이를 구현하기 위한 구현으로, 기존의 자바스크립트 alert 알림이 아닌,
표준화된 모달 창 컴포넌트를 구현하고, 필요한 내용을 넣어 출력하는 식으로 가야 한다.

4. 프로그램의 실행 방식

현재는:

 - 코어 대몬이 IPC 서버를 열고
 - 해당 서버에 일렉트론과 디스코드 봇 등이 접속하는 구조

이렇게 되어있는데, 유저 경험을 위해서라면
 - 먼저 GUI를 실행
 - 그리고 백그라운드에서 코어 대몬 실행 및 모듈 로드 등

이런 식으로 가는 게 맞을 듯하다.
